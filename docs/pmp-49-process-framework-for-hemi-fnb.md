# PMBOK 6th Edition 49-Process Framework
## Hemi Steak & Seafood Grill FnB Application

This document tailors the **49 PMP processes** to the Hemi FnB implementation project.

---

## 1) Integration Management (7 processes)

| # | Process | Project-Specific Application | Primary Deliverable |
|---|---|---|---|
| 1 | Develop Project Charter | Formally authorize the Hemi FnB application project, success criteria, budget guardrails, and sponsor authority. | Project Charter |
| 2 | Develop Project Management Plan | Consolidate all subsidiary plans (scope, schedule, cost, quality, resource, comms, risk, procurement, stakeholder). | Integrated Project Management Plan |
| 3 | Direct and Manage Project Work | Execute sprint/release work for reservation, POS, kitchen, inventory, and reporting modules. | Work Performance Data + Deliverables |
| 4 | Manage Project Knowledge | Capture lessons from pilot rollout, service-hour issues, and kitchen/front-of-house workflow adjustments. | Lessons Learned Register |
| 5 | Monitor and Control Project Work | Track planned vs actual progress, defect rates, and release readiness. | Work Performance Reports |
| 6 | Perform Integrated Change Control | Evaluate and approve/reject scope changes (e.g., loyalty module additions) through CCB governance. | Change Log + Approved Change Requests |
| 7 | Close Project or Phase | Obtain formal acceptance and transition operations/support ownership. | Final Acceptance + Closure Report |

## 2) Scope Management (6 processes)

| # | Process | Project-Specific Application | Primary Deliverable |
|---|---|---|---|
| 8 | Plan Scope Management | Define how FnB product and project scope will be captured, baselined, and controlled. | Scope Management Plan |
| 9 | Collect Requirements | Gather requirements from restaurant manager, chefs, servers, cashiers, finance, and IT support. | Requirements Documentation + RTM |
| 10 | Define Scope | Define in-scope capabilities: table/seat management, order flow, menu config, billing, audit trail, reporting. | Project Scope Statement |
| 11 | Create WBS | Decompose work into deliverable packages (core platform, integrations, testing, training, rollout). | WBS + WBS Dictionary |
| 12 | Validate Scope | Conduct stakeholder demos and formal acceptance for each major module. | Accepted Deliverables |
| 13 | Control Scope | Prevent unauthorized feature creep and maintain baseline integrity. | Scope Performance Information |
